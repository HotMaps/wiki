dodaju se u proizvodnu platformu novim izdanjem </li></ol><p><img alt="integracija podataka" src="en-images/data-integration-workflow.png"/></p><p> Ako skup podataka ne uspije tijekom integracije, Ovdje je GitLab organizacija u koju treba gurnuti <a href="https://gitlab.com/hotmaps">skupove podataka</a> : <a href="https://gitlab.com/hotmaps">skupovi podataka na GitLabu</a> . </p><p> Jednom dnevno se skladišta provjeravaju za nove obveze i integriraju ako jesu. Proces integracije provjerava jesu li podaci u skladu s specifikacijom ili ne. </p><p> Evo specifikacije: <a href="uploads/Hotmaps_Data-upload-on-Gitlab_2017-12-04_V4.pdf">Hotmaps_Data-upload-on-Gitlab_2017-12-04_V4.pdf</a> </p><p><ins> <code><strong><a href="#table-of-contents">To Top</a></strong></code> </ins> </p><h2> <a class="anchor" id="list-of-main-repositories" href="#list-of-main-repositories"><i class="fa fa-link"></i></a> Popis glavnih spremišta </h2><p> Kod aplikacije nalazi se na GitHubu pod <a href="https://github.com/HotMaps">organizacijom Hotmaps</a> . Ova organizacija posjeduje nekoliko spremišta </p><ul><li> <a href="https://github.com/HotMaps/Hotmaps-toolbox-service">Hotmaps-toolbox-klijent</a> sadrži <a href="https://github.com/HotMaps/Hotmaps-toolbox-service">suštinu</a> naše aplikacije. To je kutni projekt (JavaScript) </li><li> <a href="https://github.com/HotMaps/Hotmaps-toolbox-service">Hotmaps-toolbox-usluga</a> sadrži API za našu aplikaciju. Temelji se na Flask (Python) </li><li> <a href="https://github.com/HotMaps/hotmaps_wiki">Hotmaps-wiki</a> je Wiki koji trenutno čitate </li><li> <a href="https://github.com/HotMaps/base_calculation_module">modul</a> za osnovni <a href="https://github.com/HotMaps/base_calculation_module">proračun</a> je osnovni predložak koji možete koristiti za stvaranje vlastitih modula za proračun Hotmapova </li><li> popis modula za proračun </li></ul><p><ins> <code><strong><a href="#table-of-contents">To Top</a></strong></code> </ins> </p><h2> <a class="anchor" id="how-to-contribute-code" href="#how-to-contribute-code"><i class="fa fa-link"></i></a> Kako pridonijeti kod </h2><p> Ako želite dodati neki kôd Hotmapsu, imate dvije mogućnosti: ako želite ažurirati sučelje ili pomoćni prostor izravno, morate izmijeniti klijentsko ili servisno spremište alatnog okvira. Ako želite dodati vlastiti modul izračuna, možete stvoriti vlastito spremište slijedeći <a href="https://github.com/HotMaps/base_calculation_module">readme repozitorija base_calculation_module</a> </p><p> Ako želite obaviti neki posao na Git spremištu, nemojte izravno raditi s glavnom ogrankom. Izradite novu granu iz grane za razvoj, obavite svoj rad na toj grani i kad se testira vaša značajka, možete svoj rad spojiti s granom za razvoj kao što je prikazano na sljedećem grafikonu. </p><p><img alt="git_workflow" src="en-images/git_workflow.png"/></p><p> Da biste nešto gurnuli u neko spremište Hotmaps-a, morate biti član tima Hotmaps-a, ako niste, još uvijek ste u mogućnosti izvršiti vilicu našeg alata za razvoj vlastitog alata. </p><p> Više podataka o radu možete pronaći u ovim dokumentima: </p><ul><li> <a href="uploads/Hotmaps_python_best_practices_tutorial_2017-08-07_v01.pdf">Hotmaps_python_best_practices_tutorial_2017-08-07_v01.pdf</a> </li><li> <a href="uploads/Hotmaps_Testing_in_python_tutorial_pytest_2017-08-07_v01.pdf">Hotmaps_Testing_in_python_tutorial_pytest_2017-08-07_v01.pdf</a> </li><li> <a href="uploads/GitFlow_Guidelines_CREM_2017-02-01.pdf">GitFlow_Guidelines_CREM_2017-02-01.pdf</a> </li></ul><p><ins> <code><strong><a href="#table-of-contents">To Top</a></strong></code> </ins> </p><h2> <a class="anchor" id="description-of-it-infrastructure" href="#description-of-it-infrastructure"><i class="fa fa-link"></i></a> Opis IT infrastrukture </h2><p><img alt="ReverseProxy_architecture_latest" src="en-images/ReverseProxy_architecture_latest.png"/></p><p> Sve usluge i komponente koriste se kroz vlastiti Docker spremnik. Svi su ti spremnici definirani u jednoj datoteci za slanje dokumenata. Slika iznad predstavlja IT arhitekturu Hotmapa. </p><p> Neke su partnerske organizacije ograničile komunikaciju samo na luku 80. Kako bi se izbjegli problemi uzrokovani tim ograničenjem, napravljeno je obrnuto proxy poslužitelj. Ovaj obrnuti proxy nudi jednu ulaznu točku, a zatim distribuira zahtjev koji je kupac poslao dotičnoj usluzi. Obrnuti proxy sastoji se od tri komponente: </p><ol><li> Obrnuti proxy poslužitelj: služi kao jedinstvena ulazna točka i distribuira zahtjeve na prave usluge. </li><li> Proxy gen: to je usluga koja automatski preslikava sve usluge u obrnuti proxy. Stoga nije potrebno ručno dodavati novu uslugu u konfiguraciju proxyja </li><li> omogućuje-šifriranje: to je usluga koja omogućuje upotrebu SSL protokola. Potrebno je za aktiviranje https protokola. SSL certifikate potpisuje adresa e-pošte konfigurirana u ovoj usluzi. </li></ol><p> Postoje tri mreže: </p><ul><li> hotmaps_nginx omogućava obrnutim proxyju da komunicira s api, frontendom i geoserverom. Omogućuje uglavnom distribuciju zahtjeva u tri odgovarajuće službe. </li><li> hotmaps_backend omogućava komunikaciju između svih komponenti podupirača: api, frontend, geoserver i PostgreSQL baze podataka. </li><li> hotmaps_cm-net omogućava komunikaciju između svakog proračunskog modula i api. </li></ul><p> Svaki modul za proračun ima svoj Docker spremnik. </p><h3> <a class="anchor" id="run-with-docker" href="#run-with-docker"><i class="fa fa-link"></i></a> Trči s Dockerom </h3><p> Hotmaps koristi <a href="https://www.docker.com/">Docker</a> softver i <a href="https://docs.docker.com/compose/">Docker-Compose</a> alat za upravljanje spremnicima. Datoteka docker-compose.yml sadrži cijelu konfiguraciju Docker arhitekture (konfiguracija spremnika, mreža, veza, ...). To omogućuje pokretanje spremnika s jednom jednostavnom naredbom: </p><pre> <code class="language-shell">docker-compose up</code> </pre><p> <em>Na web-lokaciji Docker nalazi se više o docker-compose: <a href="https://docs.docker.com/compose/reference/">Sastavi referencu naredbenog retka</a> i <a href="https://docs.docker.com/compose/compose-file/">Sastavi referencu datoteke</a> .</em> </p><p> Postoji samo jedan spremnik koji se pokreće odvojeno od ostalih: to je baza podataka, jer treba stalno biti u tijeku. Zbog toga se ne nalazi u konfiguracijskoj datoteci docker-compose. </p><h3> <a class="anchor" id="server-infrastructure" href="#server-infrastructure"><i class="fa fa-link"></i></a> Infrastrukturna poslužiteljica </h3><h4> <a class="anchor" id="infrastructure" href="#infrastructure"><i class="fa fa-link"></i></a> Infrastruktura </h4><p> Trenutno je poslužitelj domaćin u HES-SO u Švicarskoj.
