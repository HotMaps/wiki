Top</code></strong></a></p><h2><a class="anchor" id="dataset-integration" href="#dataset-integration"><i class="fa fa-link"></i></a> Integracija skupa podataka</h2><p> Integracija novih javnih skupova podataka obrađuje se na sljedeći način:</p><ol><li> skupovi podataka moraju se gurnuti u git spremište ( <a href="#add-your-datasets-on-gitlab">dodajte svoje skupove podataka na GitLab</a> )</li><li> svake večeri skripta integrira nove / ažurirane skupove podataka u DEV platformu</li><li> ako je sve funkcioniralo dobro, skup podataka sada je dostupan na DEV platformi i programeri ga mogu integrirati u svoj kod</li><li> Nakon što je kodiranje završeno, nove značajke dodaju se na produkcijsku platformu kroz novo izdanje</li></ol><p> <strong>Integracija podataka</strong></p><p> Ako skup podataka zakaže tijekom integracije, na Taigi (platforma za upravljanje projektima) stvara se problem. Problem prikazuje pogrešku koja se pojavila i programer bi je trebao ispraviti i ponovno poslati svoj rad u Git kako bi ga skripta pokušala integrirati sljedeće večeri.</p><p> Izvorni kod integracijske skripte dostupan je na ovoj poveznici: <a href="https://github.com/HotMaps/CI_DatasetIntegration">Integracija podataka</a></p><p> <a href="#table-of-contents"><strong><code>To Top</code></strong></a></p><h2><a class="anchor" id="add-your-datasets-on-gitlab" href="#add-your-datasets-on-gitlab"><i class="fa fa-link"></i></a> Dodajte svoje skupove podataka na GitLab</h2><p> Da biste dodali skupove podataka na popis javnih skupova podataka, moraju se gurnuti u novo Git spremište na GitLabu. Evo GitLab organizacije u koju treba gurnuti <a href="https://gitlab.com/hotmaps">skupove podataka</a> : <a href="https://gitlab.com/hotmaps">Skupovi podataka na GitLabu</a> .</p><p> Jednom dnevno spremišta provjeravaju se ima li novih predavanja i integriraju se ako jesu. Proces integracije provjerava odgovaraju li podaci specifikaciji ili ne.</p><p> Evo specifikacije: <a href="https://wiki.hotmaps.eu/uploads/Hotmaps_Data-upload-on-Gitlab_2017-12-04_V4.pdf">Hotmaps_Data-upload-on-Gitlab_2017-12-04_V4.pdf</a></p><p> <a href="#table-of-contents"><strong><code>To Top</code></strong></a></p><h2><a class="anchor" id="list-of-main-repositories" href="#list-of-main-repositories"><i class="fa fa-link"></i></a> Popis glavnih spremišta</h2><p> Kôd aplikacije nalazi se na GitHubu pod <a href="https://github.com/HotMaps">organizacijom Hotmaps</a> . Ova organizacija posjeduje nekoliko spremišta</p><ul><li> <a href="https://github.com/HotMaps/Hotmaps-toolbox-service">Hotmaps-toolbox-client</a> sadrži <a href="https://github.com/HotMaps/Hotmaps-toolbox-service">prednju</a> stranu naše aplikacije. To je kutni projekt (JavaScript)</li><li> <a href="https://github.com/HotMaps/Hotmaps-toolbox-service">Hotmaps-toolbox-service</a> sadrži API za našu aplikaciju. Temelji se na tikvici (Python)</li><li> <a href="https://github.com/HotMaps/base_calculation_module">osnovni-kalkulator-modul</a> osnovni je predložak koji možete koristiti za stvaranje vlastitih modula izračuna za Hotmaps</li><li> popis modula za proračun</li></ul><p> <a href="#table-of-contents"><strong><code>To Top</code></strong></a></p><h2><a class="anchor" id="how-to-contribute-code" href="#how-to-contribute-code"><i class="fa fa-link"></i></a> Kako dodati kôd</h2><p> Ako želite dodati neki kôd na Hotmaps, imate dvije mogućnosti: ako želite izravno ažurirati sučelje ili pozadinu, morate izmijeniti spremište klijenta ili usluge alata. Ako želite dodati vlastiti modul za izračun, možete stvoriti vlastito spremište slijedeći <a href="https://github.com/HotMaps/base_calculation_module">readme spremišta base_calculation_module</a></p><p> Ako želite obaviti neki posao na Git spremištu, nemojte raditi izravno s glavnom granom. Stvorite novu granu od grane razvijanja, obavite svoj posao na ovoj grani i kada se testira vaša značajka, možete spojiti svoj rad s granom razvijanja, kao što je prikazano na sljedećem grafikonu.</p><p> <strong>Git_workflow</strong></p><p> Da biste nešto gurnuli u neko spremište Hotmaps, morate biti član Hotmaps tima, ako niste, još uvijek možete izvršiti račvanje našeg alata za razvoj vlastitog alata.</p><p> Više informacija o načinu rada možete pronaći u ovim dokumentima:</p><ul><li> <a href="https://wiki.hotmaps.eu/uploads/Hotmaps_python_best_practices_tutorial_2017-08-07_v01.pdf">Hotmaps_python_best_practices_tutorial_2017-08-07_v01.pdf</a></li><li> <a href="https://wiki.hotmaps.eu/uploads/Hotmaps_Testing_in_python_tutorial_pytest_2017-08-07_v01.pdf">Hotmaps_Testing_in_python_tutorial_pytest_2017-08-07_v01.pdf</a></li><li> <a href="https://wiki.hotmaps.eu/uploads/GitFlow_Guidelines_CREM_2017-02-01.pdf">GitFlow_Guidelines_CREM_2017-02-01.pdf</a></li></ul><p> <a href="#table-of-contents"><strong><code>To Top</code></strong></a></p><h2><a class="anchor" id="description-of-it-infrastructure" href="#description-of-it-infrastructure"><i class="fa fa-link"></i></a> Opis IT infrastrukture</h2><p> <strong>ReverseProxy_architecture_latest</strong></p><p> Sve usluge i komponente koriste se kroz vlastiti Docker spremnik. Svi su ti spremnici definirani u jednoj datoteci za sastavljanje dokerskih datoteka. Gornja slika predstavlja IT arhitekturu Hotmapsa.</p><p> Neke partnerske organizacije ograničile su komunikaciju samo na priključak 80. Da bi se izbjegli problemi uzrokovani ovim ograničenjem, napravljeno je stvaranje obrnutog proxyja. Ovaj obrnuti proxy nudi jednu ulaznu točku, a zatim distribuira zahtjev koji je kupac poslao dotičnoj usluzi. Obrnuti proxy sastoji se od tri komponente:</p><ol><li> Obrnuti proxy poslužitelj: služi kao jedinstvena ulazna točka i distribuira zahtjeve pravim službama.</li><li> Proxy-gen: usluga je koja automatski preslikava sve usluge u obrnutom proxyju. Stoga nije potrebno ručno dodavati novu uslugu u konfiguraciju proxyja</li><li> lets-encrypt: to je usluga koja omogućuje upotrebu SSL protokola. Potrebno je kako bi se aktivirao https protokol. SSL certifikati potpisani su adresom e-pošte konfiguriranom u ovoj usluzi.</li></ol><p> Postoje tri mreže:</p><ul><li> hotmaps_nginx omogućuje obrnutom proxyju da komunicira s API-jem, sučeljem i GeoServerom. Uglavnom omogućuje distribuciju zahtjeva ispravnoj službi među trojicom.</li><li> hotmaps_backend omogućuje komunikaciju između svih komponenti pozadine: API-ja, sučelja, GeoServera i baze podataka PostgreSQL.</li><li> hotmaps_cm-net omogućuje komunikaciju između svakog modula za izračunavanje i API-ja.</li></ul><p> Svaki modul za izračun ima svoj Docker spremnik.</p><p> <a href="#table-of-contents"><strong><code>To Top</code></strong></a></p><h3><a class="anchor" id="run-with-docker" href="#run-with-docker"><i class="fa fa-link"></i></a> Trči s Dockerom</h3><p> Hotmaps koristi <a href="https://www.docker.com/">Docker</a> softver i <a href="https://docs.docker.com/compose/">Docker-Compose</a> alat za upravljanje spremnicima. Datoteka docker-compose.yml sadrži cijelu konfiguraciju Dockerove arhitekture (konfiguracija spremnika, mreža, veza, ...). To omogućuje pokretanje spremnika jednom jednostavnom naredbom:</p><pre> <code class="language-shell">docker-compose up</code></pre><p> <em>Na web mjestu Dockera nalazi se više informacija o docker-compose: <a href="https://docs.docker.com/compose/reference/">Sastavi referencu naredbenog retka</a> i <a href="https://docs.docker.com/compose/compose-file/">Sastavi referencu datoteke</a> .</em></p><p> Postoji samo jedan spremnik koji se pokreće odvojeno od ostalih: to je baza podataka, jer ona mora stalno raditi. Zbog toga nije u konfiguracijskoj datoteci docker-compose.</p><p> <a href="#table-of-contents"><strong><code>To Top</code></strong></a></p><h3><a class="anchor" id="server-infrastructure" href="#server-infrastructure"><i class="fa fa-link"></i></a> Poslužiteljska infrastruktura</h3><h4><a class="anchor" id="infrastructure" href="#infrastructure"><i class="fa fa-link"></i></a> Infrastruktura</h4><p> Trenutno je poslužitelj hostiran na HES-SO u Švicarskoj. Na raspolaganju su 2 stroja: jedan za razvoj (razvoj i ispitivanje) i jedan za proizvodnju (stvarni <a href="https://www.hotmaps.eu">set</a> alata dostupan na <a href="https://www.hotmaps.eu">www.hotmaps.eu</a> ).</p><p> Oba stroja imaju iste specifikacije:</p><ul><li> CPU: Intel Xeon E5-2680 v4 (8) @ 2,4 GHz)</li><li> RAM: 16 GB</li><li> HDD: 500 GB</li><li> OS: Ubutnu 16.04 LTS</li></ul><h4><a class="anchor" id="performance" href="#performance"><i class="fa fa-link"></i></a> Izvođenje</h4><p> Često izvodimo testove performansi na razvojnom poslužitelju kako bismo zajamčili određenu količinu istovremenih korisnika.</p><p> Kao primjer, dolje su rezultati prvog beta izdanja naspram budućih testova izdanja.
