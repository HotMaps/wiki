Procesul de integrare verifică dacă datele sunt conforme cu specificația sau nu. </p><p> Iată specificația: <a href="uploads/Hotmaps_Data-upload-on-Gitlab_2017-12-04_V4.pdf">Hotmaps_Data-upload-on-Gitlab_2017-12-04_V4.pdf</a> </p><p><ins> <code><strong><a href="#table-of-contents">To Top</a></strong></code> </ins> </p><h2> <a class="anchor" id="list-of-main-repositories" href="#list-of-main-repositories"><i class="fa fa-link"></i></a> Lista principalelor depozite </h2><p> Codul aplicației este localizat pe GitHub sub <a href="https://github.com/HotMaps">organizația Hotmaps</a> . Această organizație deține mai multe depozite </p><ul><li> <a href="https://github.com/HotMaps/Hotmaps-toolbox-service">Hotmaps-toolbox-client</a> conține frontendul aplicației noastre. Este un proiect unghiular (JavaScript) </li><li> <a href="https://github.com/HotMaps/Hotmaps-toolbox-service">Hotmaps-toolbox-service</a> conține API-ul pentru aplicația noastră. Se bazează pe Flask (Python) </li><li> <a href="https://github.com/HotMaps/hotmaps_wiki">Hotmaps-wiki</a> este Wiki pe care îl citiți în prezent </li><li> <a href="https://github.com/HotMaps/base_calculation_module">baza de calcul-modul</a> este șablonul de bază pe care îl puteți utiliza pentru a crea propriile module de calcul pentru Hotmaps </li><li> o listă de module de calcul </li></ul><p><ins> <code><strong><a href="#table-of-contents">To Top</a></strong></code> </ins> </p><h2> <a class="anchor" id="how-to-contribute-code" href="#how-to-contribute-code"><i class="fa fa-link"></i></a> Cum să contribuiți la cod </h2><p> Dacă doriți să adăugați unele coduri la Hotmap-uri, aveți două posibilități: dacă doriți să actualizați direct interfața sau backend-ul, trebuie să modificați clientul sau depozitul de servicii al casetei de instrumente. Dacă doriți să adăugați propriul modul de calcul, puteți să vă creați propriul depozit urmând <a href="https://github.com/HotMaps/base_calculation_module">remezarea depozitului base_calculation_module</a> </p><p> Dacă doriți să efectuați unele lucrări în depozitul Git, vă rugăm să nu lucrați direct cu sucursala principală. Creați o ramură nouă din ramura dezvoltată, faceți-vă lucrările pe această ramură și când testarea funcției dvs. vă poate îmbina munca cu ramura dezvoltată așa cum se arată în graficul următor. </p><p><img alt="git_workflow" src="en-images/git_workflow.png"/></p><p> Pentru a împinge ceva către un depozit Hotmaps, trebuie să fiți membru al echipei Hotmaps, dacă nu sunteți încă, sunteți în stare să efectuați o furculiță a instrumentului nostru pentru a vă dezvolta propriul instrument. </p><p> Puteți găsi mai multe informații despre modul de lucru în aceste documente: </p><ul><li> <a href="uploads/Hotmaps_python_best_practices_tutorial_2017-08-07_v01.pdf">Hotmaps_python_best_practices_tutorial_2017-08-07_v01.pdf</a> </li><li> <a href="uploads/Hotmaps_Testing_in_python_tutorial_pytest_2017-08-07_v01.pdf">Hotmaps_Testing_in_python_tutorial_pytest_2017-08-07_v01.pdf</a> </li><li> <a href="uploads/GitFlow_Guidelines_CREM_2017-02-01.pdf">GitFlow_Guidelines_CREM_2017-02-01.pdf</a> </li></ul><p><ins> <code><strong><a href="#table-of-contents">To Top</a></strong></code> </ins> </p><h2> <a class="anchor" id="description-of-it-infrastructure" href="#description-of-it-infrastructure"><i class="fa fa-link"></i></a> Descrierea infrastructurii IT </h2><p><img alt="ReverseProxy_architecture_latest" src="en-images/ReverseProxy_architecture_latest.png"/></p><p> Toate serviciile și componentele sunt utilizate prin propriul container Docker. Toate aceste containere sunt definite într-un singur fișier de tip docker-compose. Imaginea de mai sus reprezintă arhitectura IT a Hotmaps-urilor. </p><p> Unele organizații partenere au limitat doar comunicarea la portul 80. Pentru a evita problemele cauzate de această limitare, s-a făcut crearea unui proxy invers. Acest proxy invers oferă un singur punct de intrare și apoi distribuie solicitarea trimisă de client serviciului în cauză. Proxy invers este compus din trei componente: </p><ol><li> Server proxy invers: servește ca punct de intrare unic și distribuie cererile către serviciile potrivite. </li><li> Proxy-gen: este un serviciu care mapează automat toate serviciile din proxy-ul invers. Astfel, nu este necesară adăugarea manuală a unui nou serviciu la configurația proxy </li><li> Lasă-criptă: este un serviciu care permite utilizarea protocolului SSL. Pentru a activa protocolul https este necesar. Certificatele SSL sunt semnate printr-o adresă de e-mail configurată în acest serviciu. </li></ol><p> Există trei rețele: </p><ul><li> hotmaps_nginx permite proxy-ului invers să comunice cu api, frontend și geoserver. Permite în principal distribuirea unei solicitări către serviciul corect între cele trei. </li><li> hotmaps_backend permite comunicarea între toate componentele backend-ului: api, frontend, geoserver și baza de date PostgreSQL. </li><li> hotmaps_cm-net permite comunicarea între fiecare modul de calcul și api. </li></ul><p> Fiecare modul de calcul are propriul său container Docker. </p><h3> <a class="anchor" id="run-with-docker" href="#run-with-docker"><i class="fa fa-link"></i></a> Fugi cu Docker </h3><p> Hotmaps utilizează <a href="https://www.docker.com/">Docker</a> software și <a href="https://docs.docker.com/compose/">Docker-Compose</a> instrument pentru a gestiona containere. Un fișier docker-compose.yml conține întreaga configurație a arhitecturii Docker (configurare de containere, rețele, legături, ...). Aceasta permite rularea containerelor cu o singură comandă: </p><pre> <code class="language-shell">docker-compose up</code> </pre><p> <em>Există mai multe despre docker-compose pe site-ul Docker: <a href="https://docs.docker.com/compose/reference/">Compune referința liniei de comandă</a> și <a href="https://docs.docker.com/compose/compose-file/">Compose file file</a> .</em> </p><p> Există un singur container care este rulat separat de alții: este baza de date deoarece trebuie să rămână mereu la curent. De aceea nu se află în fișierul de configurare docker-compose. </p><h3> <a class="anchor" id="server-infrastructure" href="#server-infrastructure"><i class="fa fa-link"></i></a> Infrastructura serverului </h3><h4> <a class="anchor" id="infrastructure" href="#infrastructure"><i class="fa fa-link"></i></a> Infrastructură </h4><p> Pentru moment, serverul este găzduit la HES-SO din Elveția.
