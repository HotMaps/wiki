Top</code></strong></a></p><h2><a class="anchor" id="dataset-integration" href="#dataset-integration"><i class="fa fa-link"></i></a> Integrarea setului de date</h2><p> Integrarea noilor seturi de date publice este gestionată după cum urmează:</p><ol><li> seturile de date trebuie împinse într-un depozit git ( <a href="#add-your-datasets-on-gitlab">Adăugați seturile de date pe GitLab</a> )</li><li> în fiecare seară, un script integrează seturile de date noi / actualizate pe platforma DEV</li><li> dacă totul a funcționat bine, setul de date este acum disponibil pe platforma DEV, iar dezvoltatorii îl pot integra în codul lor</li><li> Odată ce codarea este terminată, noile caracteristici sunt adăugate pe platforma de producție printr-o nouă versiune</li></ol><p> <strong>Integrarea datelor</strong></p><p> Dacă un set de date nu reușește în timpul integrării, se creează o problemă pe Taiga (platforma de gestionare a proiectelor). Problema arată eroarea apărută, iar dezvoltatorul ar trebui să o remedieze și să-și împingă din nou lucrarea către Git, astfel încât scriptul să poată încerca să-l integreze din nou în seara următoare.</p><p> Codul sursă al scriptului de integrare este disponibil la acest link: <a href="https://github.com/HotMaps/CI_DatasetIntegration">Integrare date</a></p><p> <a href="#table-of-contents"><strong><code>To Top</code></strong></a></p><h2><a class="anchor" id="add-your-datasets-on-gitlab" href="#add-your-datasets-on-gitlab"><i class="fa fa-link"></i></a> Adăugați seturile de date pe GitLab</h2><p> Pentru a adăuga seturi de date pe lista de seturi de date publice, acestea trebuie să fie împinse într-un nou depozit Git pe GitLab. Iată organizația GitLab în care seturile de date ar trebui împinse: <a href="https://gitlab.com/hotmaps">Seturi de date pe GitLab</a> .</p><p> O dată pe zi, depozitele sunt verificate pentru confirmări noi și integrate, dacă da. Procesul de integrare verifică dacă datele sunt conforme cu specificația sau nu.</p><p> Iată specificația: <a href="https://wiki.hotmaps.eu/uploads/Hotmaps_Data-upload-on-Gitlab_2017-12-04_V4.pdf">Hotmaps_Data-upload-on-Gitlab_2017-12-04_V4.pdf</a></p><p> <a href="#table-of-contents"><strong><code>To Top</code></strong></a></p><h2><a class="anchor" id="list-of-main-repositories" href="#list-of-main-repositories"><i class="fa fa-link"></i></a> Lista principalelor depozite</h2><p> Codul aplicației se află pe GitHub în cadrul <a href="https://github.com/HotMaps">organizației Hotmaps</a> . Această organizație deține mai multe depozite</p><ul><li> <a href="https://github.com/HotMaps/Hotmaps-toolbox-service">Hotmaps-toolbox-client</a> conține frontendul aplicației noastre. Este un proiect Angular (JavaScript)</li><li> <a href="https://github.com/HotMaps/Hotmaps-toolbox-service">Hotmaps-toolbox-service</a> conține API-ul pentru aplicația noastră. Se bazează pe Flask (Python)</li><li> <a href="https://github.com/HotMaps/base_calculation_module">modulul de calcul de</a> bază este șablonul de bază pe care îl puteți utiliza pentru a crea propriile module de calcul pentru Hotmaps</li><li> o listă de module de calcule</li></ul><p> <a href="#table-of-contents"><strong><code>To Top</code></strong></a></p><h2><a class="anchor" id="how-to-contribute-code" href="#how-to-contribute-code"><i class="fa fa-link"></i></a> Cum să contribuiți la cod</h2><p> Dacă doriți să adăugați un cod la Hotmaps aveți două posibilități: dacă doriți să actualizați interfața sau backend-ul direct, trebuie să modificați clientul sau depozitul de servicii al casetei de instrumente. Dacă doriți să adăugați propriul modul de calcul, vă puteți crea propriul depozit urmând <a href="https://github.com/HotMaps/base_calculation_module">readme-ul depozitului base_calculation_module</a></p><p> Dacă doriți să efectuați unele lucrări pe depozitul Git, vă rugăm să nu lucrați direct cu filiala principală. Creați o ramură nouă din ramura de dezvoltare, faceți-vă munca pe această ramură și când caracteristica dvs. este testată, puteți îmbina munca cu ramura de dezvoltare așa cum se arată în graficul următor.</p><p> <strong>Git_workflow</strong></p><p> Pentru a trimite ceva către un depozit Hotmaps, trebuie să fiți membru al echipei Hotmaps, dacă nu sunteți, puteți efectua un instrument al instrumentului nostru pentru a vă dezvolta propriul instrument.</p><p> Puteți găsi mai multe informații despre cum să lucrați în aceste documente:</p><ul><li> <a href="https://wiki.hotmaps.eu/uploads/Hotmaps_python_best_practices_tutorial_2017-08-07_v01.pdf">Hotmaps_python_best_practices_tutorial_2017-08-07_v01.pdf</a></li><li> <a href="https://wiki.hotmaps.eu/uploads/Hotmaps_Testing_in_python_tutorial_pytest_2017-08-07_v01.pdf">Hotmaps_Testing_in_python_tutorial_pytest_2017-08-07_v01.pdf</a></li><li> <a href="https://wiki.hotmaps.eu/uploads/GitFlow_Guidelines_CREM_2017-02-01.pdf">GitFlow_Guidelines_CREM_2017-02-01.pdf</a></li></ul><p> <a href="#table-of-contents"><strong><code>To Top</code></strong></a></p><h2><a class="anchor" id="description-of-it-infrastructure" href="#description-of-it-infrastructure"><i class="fa fa-link"></i></a> Descrierea infrastructurii IT</h2><p> <strong>ReverseProxy_architecture_latest</strong></p><p> Toate serviciile și componentele sunt utilizate prin propriul container Docker. Toate aceste containere sunt definite într-un singur fișier docker-compose. Imaginea de mai sus reprezintă arhitectura IT a Hotmaps.</p><p> Unele organizații partenere au limitat comunicarea doar la portul 80. Pentru a evita problemele cauzate de această limitare, s-a făcut crearea unui proxy invers. Acest proxy invers oferă un singur punct de intrare și apoi distribuie cererea trimisă de client către serviciul în cauză. Proxy-ul invers este compus din trei componente:</p><ol><li> Server proxy invers: servește ca punct de intrare unic și distribuie cererile către serviciile potrivite.</li><li> Proxy-gen: este un serviciu care mapează automat toate serviciile în proxy invers. Astfel, nu este necesar să adăugați manual un serviciu nou la configurația proxy</li><li> permite-criptare: este un serviciu care permite utilizarea protocolului SSL. Este necesar pentru a activa protocolul https. Certificatele SSL sunt semnate de o adresă de e-mail configurată în acest serviciu.</li></ol><p> Există trei rețele:</p><ul><li> hotmaps_nginx permite ca proxy-ul invers să comunice cu API-ul, frontend-ul și GeoServer-ul. Permite în principal distribuirea unei cereri către serviciul corect între cele trei.</li><li> hotmaps_backend permite comunicarea între toate componentele backend-ului: API, frontend, GeoServer și baza de date PostgreSQL.</li><li> hotmaps_cm-net permite comunicarea între fiecare modul de calcul și API.</li></ul><p> Fiecare modul de calcul are propriul container Docker.</p><p> <a href="#table-of-contents"><strong><code>To Top</code></strong></a></p><h3><a class="anchor" id="run-with-docker" href="#run-with-docker"><i class="fa fa-link"></i></a> Aleargă cu Docker</h3><p> Hotmaps utilizează <a href="https://www.docker.com/">Docker</a> software și <a href="https://docs.docker.com/compose/">Docker-Compose</a> instrument pentru a gestiona containere. Un fișier docker-compose.yml conține întreaga configurație a arhitecturii Docker (configurarea containerelor, rețelelor, linkurilor, ...). Acest lucru permite rularea containerelor cu o singură comandă simplă:</p><pre> <code class="language-shell">docker-compose up</code></pre><p> <em>Există mai multe despre docker-compose pe site-ul Docker: <a href="https://docs.docker.com/compose/reference/">compunere referință linie de comandă</a> și <a href="https://docs.docker.com/compose/compose-file/">compunere referință fișier</a> .</em></p><p> Există un singur container care este rulat separat de altele: este baza de date, deoarece trebuie să rămână activ tot timpul. De aceea nu se află în fișierul de configurare docker-compose.</p><p> <a href="#table-of-contents"><strong><code>To Top</code></strong></a></p><h3><a class="anchor" id="server-infrastructure" href="#server-infrastructure"><i class="fa fa-link"></i></a> Infrastructura serverului</h3><h4><a class="anchor" id="infrastructure" href="#infrastructure"><i class="fa fa-link"></i></a> Infrastructură</h4><p> Pentru moment, serverul este găzduit la HES-SO din Elveția.
