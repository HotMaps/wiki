<h1> <a class="anchor" id="guidelines-for-writing-a-hotmaps-wiki-page" href="#guidelines-for-writing-a-hotmaps-wiki-page"><i class="fa fa-link"></i></a> Richtlinien zum Schreiben einer Hotmaps-Wiki-Seite </h1><h2> <a class="anchor" id="table-of-contents" href="#table-of-contents"><i class="fa fa-link"></i></a> Inhaltsverzeichnis </h2><ul><li> <a href="#introduction">Einführung</a> </li><li> <a href="#wiki-toolbar-and-sidebar">Wiki-Symbolleiste und Seitenleiste</a> </li><li> <a href="#add-figures-and-images">Fügen Sie Zahlen und Bilder hinzu</a> </li><li> <a href="#classification-of-wiki-pages">Klassifizierung von Wiki-Seiten</a> </li><li> <a href="#default-structures-of-the-wiki-pages">Standardstrukturen der Wiki-Seiten</a> <ul><li> <a href="#default-structures-of-the-wiki-pages_hotmaps-data-set-repository-structure">Repository-Struktur für Hotmaps-Datensätze</a> </li><li> <a href="#default-structures-of-the-wiki-pages_general-functionalities-of-the-toolbox">Allgemeine Funktionen der Toolbox</a> </li><li> <a href="#default-structures-of-the-wiki-pages_calculation-modules">Berechnungsmodule</a> </li><li> <a href="#default-structures-of-the-wiki-pages_how-to-apply-the-hotmaps-toolbox">So wenden Sie die Hotmaps-Toolbox an</a> </li><li> <a href="#default-structures-of-the-wiki-pages_developers-area">Entwicklerbereich</a> </li></ul></li><li> <a href="#authors-and-reviewers">Autoren und Rezensenten</a> </li><li> <a href="#license">Lizenz</a> </li><li> <a href="#acknowledgement">Wissen</a> </li></ul><h2> <a class="anchor" id="introduction" href="#introduction"><i class="fa fa-link"></i></a> Einführung </h2><p> Die Hotmaps-Wiki-Plattform bietet flexible Funktionen zum Erstellen und Bearbeiten der Wiki-Seiten. Um eine Wiki-Seite erstellen oder ändern zu können, müssen Sie bei der Wiki-Plattform angemeldet sein. Dies ist nur möglich, wenn Sie ein GitHub-Konto haben und eine Wiki-Provider-Berechtigung erhalten. Um eine Erlaubnis zu erhalten, senden Sie eine E-Mail an &quot;support (at) hotmaps.eu&quot; mit Ihrer GitHub-ID. Wir werden Ihre Anfrage dann bewerten und entsprechend beantworten. Verwenden Sie zum Anmelden die Anmeldeschaltfläche in der oberen rechten Ecke. Sobald Sie angemeldet sind, können Sie die Seiten erstellen und ändern. </p><p> <a href="#table-of-contents"><strong><code>To Top</code></strong></a> </p> <h2> <a class="anchor" id="wiki-toolbar-and-sidebar" href="#wiki-toolbar-and-sidebar"><i class="fa fa-link"></i></a> Wiki-Symbolleiste und Seitenleiste </h2><p> Oben auf jeder Seite sehen Sie die Bearbeitungssymbolleiste. Die folgende Abbildung zeigt diese Symbolleiste und erläutert kurz die Funktion der einzelnen Schaltflächen. </p><img alt="" src="/en/Guidelines-for-writing-a-Hotmaps-Wiki-page/toolbar_new.png"/><p> Die Seitenleiste ist auf allen Wiki-Seiten sichtbar. Beachten Sie, dass <strong>jede Änderung in der Seitenleiste auf alle Wiki-Seiten angewendet wird</strong> . Mit anderen Worten, die Seitenleiste ist nicht spezifisch für die Seite, die Sie bearbeiten. Um die Seitenleiste zu ändern, klicken Sie in der Wiki-Symbolleiste auf die Schaltfläche &quot;Bearbeiten&quot;. Die Änderung der Seitenleiste kann über den Abschnitt unten auf der Bearbeitungsseite vorgenommen werden (siehe folgende Abbildung). </p><img alt="" src="/en/Guidelines-for-writing-a-Hotmaps-Wiki-page/sidebar.png"/><p> Um das Bearbeitungsfenster zu vergrößern, ziehen Sie es in die untere rechte Ecke, wie in der folgenden Abbildung gezeigt. </p><img alt="" src="/en/Guidelines-for-writing-a-Hotmaps-Wiki-page/enlarge_edit_window_new.png"/> [** `To Top` **] (# Inhaltsverzeichnis) <h2> <a class="anchor" id="add-figures-and-images" href="#add-figures-and-images"><i class="fa fa-link"></i></a> Fügen Sie Zahlen und Bilder hinzu </h2><p> Um Ihrer Wiki-Seite ein Bild hinzuzufügen, haben Sie zwei Möglichkeiten: </p><p> <strong>Methode I:</strong> </p><ul><li> Laden Sie zunächst das Bild über die Schaltfläche &quot;Hochladen&quot; in der Symbolleiste in Ihre gewünschte Wiki-Seite hoch. </li><li> Notieren Sie sich den bereitgestellten Link für das hochgeladene Bild. </li><li> Starten Sie die Bearbeitung dieser Wiki-Seite und verwenden Sie den Link zum Bild. </li></ul><p> <strong>Methode II:</strong> </p><ul><li> Beginnen Sie mit der Bearbeitung Ihrer gewünschten Wiki-Seite. </li><li> Ziehen Sie das Bild per Drag &amp; Drop an die Stelle im Text, an der Sie die Abbildung anzeigen möchten. </li></ul><p> <a href="#table-of-contents"><strong><code>To Top</code></strong></a> </p> <h2> <a class="anchor" id="classification-of-wiki-pages" href="#classification-of-wiki-pages"><i class="fa fa-link"></i></a> Klassifizierung von Wiki-Seiten </h2><p> Die Hotmaps-Wiki-Seiten sind in fünf verschiedene Klassen unterteilt: </p><ul><li> Repository-Struktur für Hotmaps-Datensätze </li><li> Allgemeine Funktionen der Toolbox </li><li> Berechnungsmodule </li><li> So wenden Sie die Hotmaps-Toolbox an </li><li> Entwicklerbereich </li></ul><p> Für jede Wiki-Seitenklasse wird eine Struktur mit mehreren Abschnitten vorgeschlagen (siehe nächster Abschnitt auf dieser Seite). Die Abschnitte in <strong>Fettdruck</strong> müssen Teil der Wiki-Seite sein. Der Rest sollte gegebenenfalls hinzugefügt werden. Schauen Sie sich als Beispielseite <a href="/en/CM-District-heating-potential-areas-user-defined-thresholds">diese Wiki-Seite an</a> . </p><p> Wiki-Seiten können aufgrund eingebetteter Zahlen sehr lang werden. Um den Lesern das Navigieren auf einer Seite zu erleichtern und einfach zu finden, wonach sie suchen, fügen Sie bitte über jede Seite, die Sie schreiben, ein „ <strong>Inhaltsverzeichnis</strong> “ hinzu. Verwenden Sie außerdem eine Schaltfläche &quot; <a href="#table-of-contents"><strong><code>To Top</code></strong></a> &quot; am Ende jedes Abschnitts mit einem Hyperlink zum &quot;Inhaltsverzeichnis&quot; oben auf der Wiki-Seite. </p><p> Falls Sie Ihre eigenen Figuren verwenden sollten, erstellen Sie bitte hier zuerst einen neuen Ordner für Ihre Wiki-Seite und schieben Sie dann alle Figuren in den Ordner. Bitte vermeiden Sie auch sperrige Bilder, da das Repository nur über eine begrenzte Kapazität verfügt. </p><p> Wenn Sie mit dem Schreiben Ihrer Wiki-Seite fertig sind, bitten Sie Ihre Kollegen, Ihre Seite zu überprüfen und Ihnen Feedback zu geben. Auf diese Weise können wir die geschriebenen Fehler minimieren und sicherstellen, dass die Wiki-Seiten klar geschrieben sind. Fügen Sie den Namen des Prüfers im Abschnitt &quot;Autoren und Prüfer&quot; hinzu. Folgen Sie der gleichen Struktur wie auf der bereitgestellten Beispielseite. </p><p> <strong>HINWEIS</strong> : Die Wiki-Seiten sind normalerweise in Markdown-Sprache geschrieben (einige zusätzliche Formate sind ebenfalls möglich). Wenn Sie mit Markdown nicht vertraut sind, lesen Sie <a href="https://guides.github.com/features/mastering-markdown/">hier</a> das kurze Tutorial von GitHub. </p><p> <a href="#table-of-contents"><strong><code>To Top</code></strong></a> </p> <h2> <a class="anchor" id="default-structures-of-the-wiki-pages" href="#default-structures-of-the-wiki-pages"><i class="fa fa-link"></i></a> Standardstrukturen der Wiki-Seiten </h2><h3> <a class="anchor" id="hotmaps-data-set-repository-structure" href="#hotmaps-data-set-repository-structure"><i class="fa fa-link"></i></a> Repository-Struktur für Hotmaps-Datensätze </h3><p> Hier ist der wichtigste Punkt, alle Hotmaps-Repositorys in GitLab im Auge zu behalten und die Links auf dem neuesten Stand zu halten.
