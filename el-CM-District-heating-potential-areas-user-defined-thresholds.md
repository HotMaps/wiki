<h2> Πίνακας περιεχομένων </h2><ul><li> <a href="#introduction">Εισαγωγή</a> </li><li> <a href="#inputs-and-outputs">Είσοδοι και έξοδοι</a> </li><li> <a href="#method">Μέθοδος</a> </li><li> <a href="#GitHub-Repository-of-this-calculation-module">Αποθήκη GitHub αυτής της υπομονάδας υπολογισμού</a> </li><li> <a href="#sample-run">Εκτέλεση δείγματος</a> <ul><li> <a href="#test-run-1-default-input-values">Τρέχουσα δοκιμή 1: προεπιλεγμένες τιμές εισόδου</a> </li><li> <a href="#test-run-2-modified-input-values">Τρέχουσα δοκιμή 2: τροποποιημένες τιμές εισόδου</a> </li></ul></li><li> <a href="#references">βιβλιογραφικές αναφορές</a> </li><li> <a href="#how-to-cite">Πώς να αναφέρω</a> </li><li> <a href="#authors-and-reviewers">Συγγραφείς και κριτές</a> </li><li> <a href="#license">Αδεια</a> </li><li> <a href="#acknowledgement">Αναγνώριση</a> </li></ul><h2> Εισαγωγή </h2><p> Η ζήτηση θερμότητας διαδραματίζει σημαντικό ρόλο στον προσδιορισμό των πιθανών περιοχών τηλεθέρμανσης (DH). Για παράδειγμα, η εφαρμογή της τηλεθέρμανσης σε περιοχές με χαμηλή ζήτηση θερμότητας δεν είναι οικονομικά βιώσιμη. Από την άλλη πλευρά, ο προσδιορισμός οποιασδήποτε περιοχής με υψηλή πυκνότητα ζήτησης θερμότητας ως πιθανή περιοχή DH μπορεί επίσης να είναι ανακριβής. Μια υψηλή πυκνότητα ζήτησης θερμότητας σε μια περιοχή θα μπορούσε να οφείλεται στην παρουσία μερικών καταναλωτών με πολύ υψηλή ζήτηση θερμότητας σε αυτή την περιοχή. Αντίθετα, η χαμηλή μέση πυκνότητα ζήτησης θερμότητας μπορεί να αποτελεί ένδειξη ζωνών με πολύ χαμηλή ζήτηση θερμότητας στην επιλεγμένη περιοχή. Ο στόχος της ενότητας δυναμικού DH είναι να παράσχει μια λογική ισορροπία μεταξύ της πυκνότητας ζήτησης θερμότητας σε μια περιοχή και τις ζώνες που την αποτελούν. </p><p> Η υπομονάδα δυναμικού DH προσδιορίζει τις περιοχές DH και το αντίστοιχο δυναμικό DH βάσει της πυκνότητας ζήτησης θερμότητας. Οι πυκνότητες ζήτησης θερμότητας λαμβάνονται από το στρώμα GIS εισόδου, δηλαδή από τον <strong><a href="https://gitlab.com/hotmaps/heat/heat_tot_curr_density">Ευρωπαϊκό Χάρτη Πυκνότητας Θερμότητας (EHDM)</a></strong> , ο οποίος αναπτύχθηκε κατά τη διάρκεια του <strong><a href="https://www.hotmaps-project.eu">προγράμματος Hotmaps</a></strong> . Το EHDM είναι σε μορφή ράστερ και έχει ανάλυση ενός εκταρίου και Σύστημα αναφοράς συντεταγμένων (CRS) του " <em><em>ETRS89 / LAEA Europe - EPSG 3035</em></em> ". Τα κύτταρα στο EHDM δείχνουν τις πυκνότητες θέρμανσης σε <em><strong>MWh / ha</strong></em> . </p><p> Ως έξοδος, παρουσιάζεται ένα στρώμα GIS, τρεις δείκτες και δύο διαγράμματα. Αυτές οι εξόδους εξηγούνται λεπτομερώς στην ενότητα " <a href="#sample-run">Εκτέλεση δείγματος</a> ". Το στρώμα εξόδου δείχνει τις πιθανές περιοχές DH. Κάνοντας κλικ σε κάθε περιοχή του χάρτη, εμφανίζεται ένα παράθυρο και εμφανίζεται το δυναμικό DH που αντιστοιχεί σε αυτήν την περιοχή. Μέσα στο παράθυρο δείκτη / γραφήματος απεικονίζονται σχετικοί δείκτες και διαγράμματα σχετικά με το δυναμικό DH στην επιλεγμένη ζώνη και τα δυναμικά σε υποζώνες. </p><p><ins> <code><strong><a href="#table-of-contents">To Top</a></strong></code> </ins> </p><h2> Είσοδοι και έξοδοι </h2><p> Οι παράμετροι εισόδου και τα επίπεδα, καθώς και τα επίπεδα και οι παράμετροι εξόδου είναι οι εξής. </p><p> <strong>Τα επίπεδα εισόδου και οι παράμετροι είναι:</strong> </p><ul><li> Χάρτης πυκνότητας (από προεπιλογή παρέχεται από την εργαλειοθήκη) <ul><li> σε μορφή ράστερ (* .tif) </li><li> με ανάλυση 1 εκτάριο </li><li> πυκνότητας ζήτησης σε <em><strong>MWh / εκτάριο</strong></em> </li></ul></li><li> Ελάχιστη ζήτηση θερμότητας ανά εκτάριο [ <em><strong>MWh / ha</strong></em> ]: τιμή μεταξύ <em><em>0</em></em> και <em><em>1000</em></em> </li><li> Ελάχιστη ζήτηση θερμότητας σε μια περιοχή DH [ <em><strong>GWh / έτος</strong></em> ]: μια τιμή μεταξύ <em><em>0</em></em> και <em><em>500</em></em> </li></ul><p> <strong>Τα επίπεδα και οι παράμετροι εξόδου είναι:</strong> </p><ul><li> Περιοχές DH </li><li> DH δυναμικό σε κάθε περιοχή DH </li></ul><p><ins> <code><strong><a href="#table-of-contents">To Top</a></strong></code> </ins> </p><h2> Μέθοδος </h2><p> Το δυναμικό για DH σε μια συγκεκριμένη περιοχή μπορεί να οριστεί από τη συνολική ζήτηση θερμότητας και τη χωρική κατανομή της. Στην εργαλειοθήκη Hotmaps, η ζήτηση θέρμανσης εμφανίζεται στο EHDM με τη μορφή χάρτη raster. Οποιαδήποτε επιλογή ή περικοπή από το EHDM αποτελείται από ένα ή περισσότερα κύτταρα ενός εκταρίου. Προκειμένου να οριστούν σωστά οι πιθανές περιοχές DH, τόσο η ζήτηση θερμότητας σε κάθε κύτταρο όσο και σε μια περιοχή πρέπει να φθάσει σε ένα ορισμένο επίπεδο. Για το σημείο εκκίνησης, η εργαλειοθήκη Hotmaps προτείνει προεπιλεγμένες τιμές για αυτές τις δύο παραμέτρους. Ωστόσο, ανάλογα με τη διανομή της ζήτησης θερμότητας και επίσης την τοπική εκτίμηση, ο χρήστης Hotmaps μπορεί να τροποποιήσει αυτές τις τιμές. </p><p> Ο προσδιορισμός των περιοχών DH γίνεται σε δύο στάδια: </p><p> Στο πρώτο βήμα, όλα τα κελιά με ζήτηση θέρμανσης κάτω από την παράμετρο εισόδου για την ελάχιστη ζήτηση θέρμανσης σε εκτάριο φιλτράρονται. Εξαλείφοντας αυτά τα κύτταρα από τον χάρτη, αποκτάμε ομάδες κυττάρων που είναι προσαρτημένα μεταξύ τους. Κάθε ομάδα αυτών των συνδεδεμένων κυττάρων αποτελούν μικρές ζώνες που εδώ αναφέρονται ως "συνεκτικές περιοχές". Στο δεύτερο βήμα, υπολογίζεται η συνολική ζήτηση θερμότητας σε κάθε συνεκτική περιοχή. Για κάθε συνεκτική περιοχή, εάν η συνολική ζήτηση θερμότητας είναι υψηλότερη από την παράμετρο εισόδου για την "ελάχιστη ζήτηση θερμότητας σε μια περιοχή DH", τότε θεωρείται πιθανή περιοχή DH. </p><p> Τέλος, για τις περιοχές DH, το δυναμικό υπολογίζεται και παρουσιάζεται με τη μορφή στρώματος GIS, το οποίο μπορεί να παρατηρηθεί στην εργαλειοθήκη. </p><p> Αυτός ο κώδικας χρησιμοποιεί την έννοια των συνδεδεμένων εξαρτημάτων από τη βιβλιοθήκη επεξεργασίας εικόνων του Scipy προκειμένου να ανιχνεύσει τις πιθανές περιοχές αστικής θέρμανσης. </p><p><ins> <code><strong><a href="#table-of-contents">To Top</a></strong></code> </ins> </p><h2> Αποθήκη GitHub αυτής της υπομονάδας υπολογισμού </h2><p> <a href="https://github.com/HotMaps/dh_potential/tree/develop">Εδώ</a> εμφανίζεται η εξέλιξη της
