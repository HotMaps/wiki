href="#acknowledgement">Reconnaissance</a></li></ul><h2><a class="anchor" id="introduction" href="#introduction"><i class="fa fa-link"></i></a> introduction</h2><p> La base de données créée pour Hotmaps est un entrepôt de données spatial mis à disposition de la plate-forme à des fins de visualisation et de calcul.</p><p> Il existe différents types de données:</p><ul><li> jeux de données raster (au niveau de l&#39;hectare)</li><li> jeux de données vectorielles (régions NUTS / LAU, points d&#39;intérêt, ...)</li><li> ensembles de données non spatiales (principalement liés à NUTS / LAU et liés à des couches vectorielles existantes)</li></ul><p> Les ensembles de données sont accessibles via un serveur géospatial (GeoServer) qui fournit différents services (WMS / WFS / ...) permettant à quiconque d&#39;afficher, d&#39;inclure ou d&#39;utiliser ces ensembles de données.</p><p> Les ensembles de données peuvent être trouvés sur le <a href="https://gitlab.com/hotmaps">référentiel Gitlab</a></p><p> <a href="#table-of-contents"><strong><code>To Top</code></strong></a></p><h2><a class="anchor" id="query" href="#query"><i class="fa fa-link"></i></a> Requete</h2><p> Veuillez noter que les agrégations lors de la sélection de régions NUTS ou LAU sont pré-calculées et donc plus rapides que les agrégations de sélections d&#39;hectares, qui sont calculées à la volée.</p><p> <a href="#table-of-contents"><strong><code>To Top</code></strong></a></p><h2><a class="anchor" id="indicators" href="#indicators"><i class="fa fa-link"></i></a> Indicateurs</h2><p> Les indicateurs sont définis <strong><a href="https://github.com/HotMaps/Hotmaps-toolbox-service/blob/develop/api/app/models/indicators.py">ici</a></strong> . Selon les couches, divers paramètres statistiques tels que la valeur minimale, la valeur maximale et la valeur moyenne font partie des indicateurs. Une explication de la définition des indicateurs pour les couches intégrées dans la base de données Hotmaps peut être trouvée <strong><a href="https://github.com/HotMaps/Hotmaps-toolbox-service/blob/develop/api/app/models/INDICATORS.md">ici</a></strong> .</p><p> <a href="#table-of-contents"><strong><code>To Top</code></strong></a></p><h2><a class="anchor" id="technologies" href="#technologies"><i class="fa fa-link"></i></a> Les technologies</h2><p> Les technologies utilisées pour la base de données sont:</p><ul><li> PostgreSQL version 9.6</li><li> PostGIS version 2.3</li><li> GeoServer version 2.11.2</li></ul><p> <a href="#table-of-contents"><strong><code>To Top</code></strong></a></p><h2><a class="anchor" id="references" href="#references"><i class="fa fa-link"></i></a> Les références</h2><ul><li> Simon Pezzutto, Stefano Zambotti, Silvia Croce, Pietro Zambelli, Giulia Garegnani, Chiara Scaramuzzino, Ramón Pascual Pascuas, Alyona Zubaryeva, Franziska Haas, Dagmar Exner (EURAC), Andreas Mueller (e-think), Michael Hartner (TUW), Tobias Fleiter , Anna-Lena Klingler, Matthias Kühnbach, Pia Manz, Simon Marwitz, Matthias Rehfeldt, Jan Steinbach, Eftim Popovski (Fraunhofer ISI) Revu par Lukas Kranzl, Sara Fritz (TUW) Hotmaps Project, <strong>D2.3 WP2 Report -
