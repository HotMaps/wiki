<h2> Table des matières </h2><ul><li> <a href="#Indicators">Des indicateurs</a> <ul><li> <a href="#Simple-indicator">Indicateur simple</a> </li><li> <a href="#Cross-indicator">Indicateur croisé</a> </li></ul></li><li> <a href="#Indicator-result">Résultat indicateur</a> </li><li> <a href="#references">Références</a> </li><li> <a href="#how-to-cite">Comment citer</a> </li><li> <a href="#authors-and-reviewers">Auteurs et relecteurs</a> </li><li> <a href="#license">Licence</a> </li><li> <a href="#acknowledgement">Reconnaissance</a> </li></ul><h2> Des indicateurs </h2><p> Cette documentation explique comment gérer les indicateurs de la boîte à outils Hotmaps. Un indicateur est une valeur affichée dans la boîte à outils Hotmaps du côté client. L&#39;indicateur fait partie d&#39;une couche. Indicateur est une valeur, pour une région spatiale spécifique, avec une unité. Un indicateur fait partie d&#39;un dictionnaire de couches. Un objet couche contient un tableau nommé &quot;indicateurs&quot;. Ce tableau servira à répertorier tous les indicateurs que vous souhaitez voir pour la couche. </p><pre> <code>layers = { &#39;heat_tot_curr_density_tif&#39;:{ &#39;tablename&#39;:&#39;heat_tot_curr_density_tif&#39;, &#39;from_indicator_name&#39;:&#39;stat_heat_tot_curr_density_tif&#39;, &#39;schema_scalelvl&#39;: &#39;stat&#39;, &#39;schema_hectare&#39;: &#39;geo&#39;, &#39;crs&#39;: &#39;3035&#39;, &#39;geo_column&#39;: &#39;geometry&#39;, &#39;table_type&#39;:&#39;raster&#39;, &#39;data_lvl&#39;:[&#39;NUTS 0&#39;,&#39;NUTS 1&#39;,&#39;NUTS 2&#39;,&#39;NUTS 3&#39;,&#39;LAU 2&#39;,&#39;Hectare&#39;], &#39;data_aggregated&#39;:True, &#39;scalelvl_column&#39;:&#39;&#39;, &#39;indicators&#39;:[ {&#39;table_column&#39;: &#39;sum&#39;, &#39;unit&#39;: &#39;MWh&#39;,&#39;indicator_id&#39;:&#39;consumption&#39;}, {&#39;table_column&#39;: &#39;count&#39;, &#39;unit&#39;: &#39;cells&#39;,&#39;indicator_id&#39;:&#39;count_cell&#39;}, { &#39;reference_indicator_id_1&#39;: &#39;consumption&#39;, &#39;reference_tablename_indicator_id_1&#39;: &#39;heat_tot_curr_density_tif&#39;, &#39;operator&#39;: &#39;/&#39;, &#39;reference_indicator_id_2&#39;: &#39;count_cell&#39;, &#39;reference_tablename_indicator_id_2&#39;: &#39;pop_tot_curr_density_tif&#39;, &#39;unit&#39;:&#39;MWh/person&#39;, &#39;indicator_id&#39;: &#39;heat_tot_curr_density_tif_per_pop_tot_curr_density_tif&#39; } ] } }</code> </pre><ul><li> &#39;nom de la table&#39; </li></ul><p> Nom de la table de base de données. (Exemple: &#39;heat_tot_curr_density_tif&#39;) </p><ul><li> &#39;from_indicator_name&#39; </li></ul><p> Nom du sous-tableau pour la sélection des indicateurs. <strong>Doit être unique.</strong> (Exemple: &#39;stat_heat_tot_curr_density_tif&#39;) </p><ul><li> &#39;data_aggregated&#39; </li></ul><p> Les données sont-elles agrégées ou non (Valeurs: True ou False) </p><ul><li> &#39;scalelvl_column&#39; </li></ul><p> Nom de colonne de niveau d&#39;échelle s&#39;il est différent du nom par défaut (Exemple: &#39;code&#39;) </p><ul><li> &#39;data_lvl&#39; </li></ul><p> Niveaux disponibles pour les données dans la base de données </p><ul><li> &#39;schema_scalelvl&#39; </li></ul><p> Emplacement du schéma de table pour le niveau nut. (Exemple: &#39;geo&#39;, &#39;stat&#39;, &#39;public&#39;) </p><ul><li> &#39;schema_hectare&#39; </li></ul><p> Emplacement du schéma de table pour le niveau d&#39;hectare. (Exemple: &#39;geo&#39;, &#39;stat&#39;, &#39;public&#39;) </p><ul><li> &#39;crs&#39; </li></ul><p> Projection de la géométrie (Exemple: &#39;3035&#39;, &#39;4326&#39;, &#39;4258&#39;) </p><ul><li> &#39;geo_column&#39; </li></ul><p> Nom de la colonne de géométrie dans la base de données (Exemple: &#39;geom&#39;, &#39;geometry&#39;) </p><ul><li> &#39;type_table&#39; </li></ul><p> Type de la couche dans la base de données (valeurs: &#39;vecteur&#39; ou &#39;raster&#39;). </p><p> <em><strong>Important:</strong></em> s’il s’agit d’un raster, les colonnes disponibles sont <strong>count, sum, mean, stddev, min et max.</strong> </p><ul><li> &#39;Indicateurs&#39; </li></ul><p> Il existe 2 types d&#39;indicateurs (indicateurs simples et indicateurs croisés). </p><h3> Indicateur simple </h3><p> Un indicateur simple est un objet avec 3 paramètres. </p><pre> <code>{ &#39;table_column&#39;: &#39;count&#39;, &#39;unit&#39;: &#39;cells&#39;, &#39;indicator_id&#39;:&#39;count_cell&#39; }</code> </pre><ul><li> &#39;table_colonne&#39; </li></ul><p> Il s&#39;agit de la colonne de table sélectionnée dans la table. (Exemple: &#39;compte&#39;) </p><p><img alt="sélection de colonne" src="/api/assets/table_image.png"/></p><ul><li> &#39;unité&#39; </li></ul><p> C&#39;est l&#39;unité de l&#39;indicateur. (Exemple: &#39;cellules&#39;, &#39;MWh&#39;) </p><ul><li> &#39;indicateur_id&#39; </li></ul><p> C&#39;est l&#39;identifiant de l&#39;indicateur (comme un identifiant). Ce nom <strong>doit être unique</strong> dans le tableau d&#39;indicateurs. </p><h3> Indicateur croisé </h3><p> Un indicateur croisé est un objet avec 7 paramètres. L&#39;objectif de cet indicateur est de faire un calcul entre des indicateurs simples et un indicateur croisé. </p><pre> <code>{ &#39;reference_indicator_id_1&#39;: &#39;consumption&#39;, &#39;reference_tablename_indicator_id_1&#39;:&#39;heat_tot_curr_density_tif&#39;, &#39;operator&#39;: &#39;/&#39;, &#39;reference_indicator_id_2&#39;:&#39;count_cell&#39;, &#39;reference_tablename_indicator_id_2&#39;:&#39;pop_tot_curr_density_tif&#39;, &#39;unit&#39;:&#39;MWh/person&#39;, &#39;indicator_id&#39;:&#39;heat_tot_curr_density_tif_per_pop_tot_curr_density_tif&#39; }</code> </pre><ul><li> &#39;reference_indicator_id_1&#39; </li></ul><p> Correspond à l&#39;identifiant d&#39;un indicateur simple. Ce nom <strong>doit être défini</strong> dans le tableau d&#39;indicateurs. C&#39;est la valeur numéro 1. </p><ul><li> &#39;reference_tablename_indicator_id_1&#39; </li></ul><p> Référence de la couche nomtable qui fait référence à la valeur numéro 1. (Exemple: &#39;heat_tot_curr_density_tif&#39;) </p><ul><li> &#39;opérateur&#39; </li></ul><p> Règle de calcul à appliquer aux 2 valeurs (Valeurs: &#39;/&#39; ou &#39;*&#39; ou &#39;+&#39; ou &#39;-&#39;) </p><ul><li> &#39;reference_indicator_id_1&#39; </li></ul><p> Correspond à l&#39;identifiant d&#39;un indicateur simple. Ce nom <strong>doit être défini</strong> dans le tableau d&#39;indicateurs. C&#39;est la valeur numéro 2. </p><ul><li> &#39;reference_tablename_indicator_id_2&#39; </li></ul><p> Référence de la couche nomtable qui fait référence à la valeur numéro 2. (Exemple: &#39;pop_tot_curr_density_tif&#39;) </p><ul><li> &#39;unité&#39; </li></ul><p> C&#39;est l&#39;unité de l&#39;indicateur. (Exemple: &#39;cellules&#39;, &#39;MWh&#39;) </p><ul><li> &#39;Nom&#39; </li></ul><p> C&#39;est le nom de l&#39;indicateur (comme un identifiant). Ce nom <strong>doit être unique</strong> dans le tableau d&#39;indicateurs. </p><h5> Remarque: Pour cet exemple, le calcul ci-dessous est terminé. </h5><pre> <code>reference_indicator_id_1.reference_indicator_id_1 / reference_indicator_id_1.reference_indicator_id_1 = heat_tot_curr_density_tif.consumption / pop_tot_curr_density_tif.count_cell</code> </pre><h2> Résultat indicateur </h2><p> Les résultats des indicateurs sont les suivants: </p><pre> <code>{ &quot;values&quot;: [ { &quot;unit&quot;: &quot;MWh&quot;, &quot;name&quot;: &quot;heat_tot_curr_density_tif_consumption&quot;, &quot;value&quot;: &quot;4112030.46&quot;
