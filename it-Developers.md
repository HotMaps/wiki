Tutti questi contenitori sono definiti in un unico file comporre docker. L&#39;immagine sopra rappresenta l&#39;architettura IT di Hotmaps. </p><p> Alcune organizzazioni partner hanno limitato la comunicazione solo alla porta 80. Per evitare i problemi causati da questa limitazione, è stata creata la creazione di un proxy inverso. Questo proxy inverso offre un unico punto di accesso e quindi distribuisce la richiesta inviata dal cliente al servizio interessato. Il proxy inverso è composto da tre componenti: </p><ol><li> Server proxy inverso: funge da punto di accesso unico e distribuisce le richieste ai servizi giusti. </li><li> Proxy-gen: è un servizio che mappa automaticamente tutti i servizi nel proxy inverso. Pertanto, non è necessario aggiungere manualmente un nuovo servizio alla configurazione proxy </li><li> lets-encrypt: è un servizio che consente l&#39;uso del protocollo SSL. È necessario per attivare il protocollo https. I certificati SSL sono firmati da un indirizzo e-mail configurato in questo servizio. </li></ol><p> Esistono tre reti: </p><ul><li> hotmaps_nginx consente al proxy inverso di comunicare con l&#39;API, il frontend e il geoserver. Permette principalmente di distribuire una richiesta al servizio corretto tra i tre. </li><li> hotmaps_backend consente la comunicazione tra tutti i componenti del backend: api, frontend, geoserver e il database PostgreSQL. </li><li> hotmaps_cm-net consente la comunicazione tra ciascun modulo di calcolo e l&#39;API. </li></ul><p> Ogni modulo di calcolo ha il proprio contenitore Docker. </p><h3> Esegui con Docker </h3><p> HotMaps utilizza <a href="https://www.docker.com/">Docker</a> software e <a href="https://docs.docker.com/compose/">Docker-Compose</a> strumento per gestire i contenitori. Un file docker-compose.yml contiene l&#39;intera configurazione dell&#39;architettura Docker (configurazione di contenitori, reti, collegamenti, ...). Ciò consente di eseguire i contenitori con un semplice comando: </p><pre> <code class="language-shell">docker-compose up</code> </pre><p> <em>C&#39;è di più su docker-compose sul webiste di Docker: componi <a href="https://docs.docker.com/compose/reference/">riferimento da riga di comando</a> e <a href="https://docs.docker.com/compose/compose-file/">componi riferimento a file</a> .</em> </p><p> Esiste un solo contenitore che viene eseguito separatamente dagli altri: è il database perché deve rimanere sempre attivo. Ecco perché non si trova nel file di configurazione comporre docker. </p><h3> Infrastruttura server </h3><h4> Infrastruttura </h4><p> Per il momento, il server è ospitato presso l&#39;HES-SO in Svizzera.
