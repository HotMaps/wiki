A kérdés megmutatja a felvetett hibát, és a fejlesztőnek javítania kell, és újra be kell tolnia munkáját Gitbe, hogy a szkript másnap este megpróbálja újra integrálni.</p><p> Az integrációs szkript forráskódja ezen a linken érhető el: <a href="https://github.com/HotMaps/CI_DatasetIntegration">Adatok integrálása</a></p><p> <a href="#table-of-contents"><strong><code>To Top</code></strong></a></p><h2><a class="anchor" id="add-your-datasets-on-gitlab" href="#add-your-datasets-on-gitlab"><i class="fa fa-link"></i></a> Adja hozzá az adatkészleteket a GitLab-on</h2><p> Adatkészletek felvételéhez a nyilvános adatkészletek listájába azokat egy új Git-adattárba kell helyezni a GitLab-on. Itt van a GitLab Szervezet, ahová az adatkészleteket kell tolni: <a href="https://gitlab.com/hotmaps">Adatkészletek a GitLab-on</a> .</p><p> Naponta egyszer ellenőrzik az adattárak új elkötelezettségeit és integrálják, ha igen. Az integrációs folyamat ellenőrzi, hogy az adatok megfelelnek-e a specifikációnak.</p><p> Itt van a specifikáció: <a href="https://wiki.hotmaps.eu/uploads/Hotmaps_Data-upload-on-Gitlab_2017-12-04_V4.pdf">Hotmaps_Data-upload-on-Gitlab_2017-12-04_V4.pdf</a></p><p> <a href="#table-of-contents"><strong><code>To Top</code></strong></a></p><h2><a class="anchor" id="list-of-main-repositories" href="#list-of-main-repositories"><i class="fa fa-link"></i></a> A fő adattárak listája</h2><p> Az alkalmazás kódja a GitHub-on található a <a href="https://github.com/HotMaps">Hotmaps szervezet alatt</a> . Ennek a szervezetnek több tárolója van</p><ul><li> <a href="https://github.com/HotMaps/Hotmaps-toolbox-service">A Hotmaps-toolbox-client</a> tartalmazza az alkalmazás <a href="https://github.com/HotMaps/Hotmaps-toolbox-service">felületét</a> . Ez egy szögletes projekt (JavaScript)</li><li> <a href="https://github.com/HotMaps/Hotmaps-toolbox-service">A Hotmaps-toolbox-service</a> tartalmazza az alkalmazás API <a href="https://github.com/HotMaps/Hotmaps-toolbox-service">-ját</a> . Lombikon (Python) alapul</li><li> <a href="https://github.com/HotMaps/base_calculation_module">Az alap-számítási modul</a> az az alapvető sablon, amellyel saját számítási modulokat hozhat létre a Hotmaps számára</li><li> a számítási modulok listája</li></ul><p> <a href="#table-of-contents"><strong><code>To Top</code></strong></a></p><h2><a class="anchor" id="how-to-contribute-code" href="#how-to-contribute-code"><i class="fa fa-link"></i></a> Hogyan lehet hozzájárulni a kódhoz</h2><p> Ha néhány kódot szeretne hozzáadni a Hotmaps-hoz, akkor két lehetősége van: ha közvetlenül szeretné frissíteni a felületet vagy a háttérprogramot, akkor módosítania kell az eszköztár kliensét vagy szolgáltatástárát. Ha hozzá szeretné adni a saját számítási modulját, létrehozhatja saját <a href="https://github.com/HotMaps/base_calculation_module">tárát az base_calculation_module repository readme-jének követésével.</a></p><p> Ha valamilyen munkát szeretne végrehajtani a Git adattáron, kérjük, ne dolgozzon közvetlenül a fő ággal. Hozzon létre egy új ágat a fejlesztési ágból, végezze el a munkáját ezen az ágon, és amikor a szolgáltatása tesztelésre kerül, egyesítheti munkáját a fejlesztési ággal az alábbi grafikon szerint.</p><p> <strong>Git_workflow</strong></p><p> Ahhoz, hogy valamit eljusson valamelyik Hotmaps-adattárba, a Hotmaps csapatának tagjának kell lennie, ha nem, akkor is képes végrehajtani az eszközünk egy villáját, hogy kifejlessze saját eszközét.</p><p> Ezekről a dokumentumokról további információkat talál a munkavégzésről:</p><ul><li> <a href="https://wiki.hotmaps.eu/uploads/Hotmaps_python_best_practices_tutorial_2017-08-07_v01.pdf">Hotmaps_python_best_practices_tutorial_2017-08-07_v01.pdf</a></li><li> <a href="https://wiki.hotmaps.eu/uploads/Hotmaps_Testing_in_python_tutorial_pytest_2017-08-07_v01.pdf">Hotmaps_Testing_in_python_tutorial_pytest_2017-08-07_v01.pdf</a></li><li> <a href="https://wiki.hotmaps.eu/uploads/GitFlow_Guidelines_CREM_2017-02-01.pdf">GitFlow_Guidelines_CREM_2017-02-01.pdf</a></li></ul><p> <a href="#table-of-contents"><strong><code>To Top</code></strong></a></p><h2><a class="anchor" id="description-of-it-infrastructure" href="#description-of-it-infrastructure"><i class="fa fa-link"></i></a> Az informatikai infrastruktúra leírása</h2><p> <strong>ReverseProxy_architecture_latest</strong></p><p> Az összes szolgáltatást és alkatrészt a saját Docker konténerén keresztül használják. Ezeket a tárolókat egyetlen docker-compose fájl definiálja. A fenti kép a Hotmaps informatikai architektúráját ábrázolja.</p><p> Egyes partnerszervezetek csak a 80-as portra korlátozták a kommunikációt. Az e korlátozás okozta problémák elkerülése érdekében fordított proxyt hoztak létre. Ez a fordított proxy egyetlen belépési pontot kínál, majd elosztja az ügyfél által küldött kérést az érintett szolgáltatásnak. A fordított proxy három összetevőből áll:</p><ol><li> Fordított proxykiszolgáló: egyedi belépési pontként szolgál, és a megfelelő szolgáltatásoknak terjeszti a kéréseket.</li><li> Proxy-gen: ez egy olyan szolgáltatás, amely automatikusan feltérképezi az összes szolgáltatást a fordított proxyban. Így nem szükséges új szolgáltatást manuálisan hozzáadni a proxy konfigurációjához</li><li> lets-encrypt: ez egy olyan szolgáltatás, amely lehetővé teszi az SSL protokoll használatát. A https protokoll aktiválásához szükséges. Az SSL tanúsítványokat a szolgáltatásban konfigurált e-mail cím írja alá.</li></ol><p> Három hálózat létezik:</p><ul><li> A hotmaps_nginx lehetővé teszi, hogy a fordított proxy kommunikáljon az API-val, a kezelőfelülettel és a GeoServerrel. Elsősorban lehetővé teszi a helyes szolgáltatásra vonatkozó kérés terjesztését a három között.</li><li> A hotmaps_backend lehetővé teszi a kommunikációt a háttérprogram összes összetevője között: API, frontend, GeoServer és a PostgreSQL adatbázis.</li><li> A hotmaps_cm-net lehetővé teszi az egyes számítási modulok és az API közötti kommunikációt.</li></ul><p> Minden számítási modulnak megvan a maga Docker tárolója.</p><p> <a href="#table-of-contents"><strong><code>To Top</code></strong></a></p><h3><a class="anchor" id="run-with-docker" href="#run-with-docker"><i class="fa fa-link"></i></a> Fuss Dockerrel</h3><p> A Hotmaps a <a href="https://www.docker.com/">Docker</a> szoftvert és a <a href="https://docs.docker.com/compose/">Docker-Compose</a> eszközt használja a tárolók kezeléséhez. A docker-compose.yml fájl tartalmazza a Docker architektúra teljes konfigurációját (tárolók, hálózatok, hivatkozások konfigurációja ...). Ez lehetővé teszi a konténerek futtatását egyetlen egyszerű paranccsal:</p><pre> <code class="language-shell">docker-compose up</code></pre><p> <em>A docker-compose-ról többet megtudhat a Docker webhelyén: <a href="https://docs.docker.com/compose/reference/">Compose parancssori referencia</a> és <a href="https://docs.docker.com/compose/compose-file/">Compose file hivatkozás</a> .</em></p><p> Csak egy tároló van, amelyet külön-külön futtatnak: ez az adatbázis, mert folyamatosan fent kell maradnia. Ezért nincs benne a docker-compose konfigurációs fájlban.</p><p> <a href="#table-of-contents"><strong><code>To Top</code></strong></a></p><h3><a class="anchor" id="server-infrastructure" href="#server-infrastructure"><i class="fa fa-link"></i></a> Szerver infrastruktúra</h3><h4><a class="anchor" id="infrastructure" href="#infrastructure"><i class="fa fa-link"></i></a> Infrastruktúra</h4><p> Jelenleg a szervert a svájci HES-SO üzemelteti. 2 gép áll rendelkezésre: egy fejlesztésre (fejlesztés és tesztelés) és egy gyártásra (a tényleges eszköztár, elérhető a <a href="https://www.hotmaps.eu">www.hotmaps.eu oldalon</a> ).</p><p> Mindkét gép specifikációja megegyezik:</p><ul><li> CPU: Intel Xeon E5-2680 v4 (8) @ 2,4 GHz)</li><li> RAM: 16 GB</li><li> HDD: 500 GB</li><li> Operációs rendszer: Ubutnu 16.04 LTS</li></ul><h4><a class="anchor" id="performance" href="#performance"><i class="fa fa-link"></i></a> Teljesítmény</h4><p> Gyakran futtatunk teljesítményteszteket a fejlesztőkiszolgálón annak érdekében, hogy garantáljuk az egyidejű felhasználók bizonyos mennyiségét.</p><p> Példaként az alábbiakban bemutatjuk az első béta kiadás és a jövőbeni kiadási tesztek eredményeit.
