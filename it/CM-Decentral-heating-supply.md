<h1> <a class="anchor" id="cm-decentral-heating-supply" href="#cm-decentral-heating-supply"><i class="fa fa-link"></i></a> CM Mandata riscaldamento decentralizzata </h1><h2> <a class="anchor" id="table-of-contents" href="#table-of-contents"><i class="fa fa-link"></i></a> Sommario </h2><ul><li> <a href="#in-a-glance">In uno sguardo</a> </li><li> <a href="#introduction">introduzione</a> </li><li> <a href="#inputs-and-outputs">Ingressi e uscite</a> <ul><li> <a href="#inputs-and-outputs_inputs">Ingressi</a> <ul><li> <a href="#inputs-and-outputs_inputs_default-input-data">Dati di input predefiniti</a> </li><li> <a href="#inputs-and-outputs_inputs_manual-input-data">Dati di input manuale</a> </li></ul></li><li> <a href="#inputs-and-outputs_outputs">Uscite</a> <ul><li> <a href="#inputs-and-outputs_outputs_indicators">Indicatori</a> </li><li> <a href="#inputs-and-outputs_outputs_graphs">Grafici</a> </li></ul></li></ul></li><li> <a href="#methodology">Metodologia</a> </li><li> <a href="#github-repository-of-this-calculation-module">Repository GitHub di questo modulo di calcolo</a> </li><li> <a href="#sample-run">Esempio di corsa</a> <ul><li> <a href="#sample-run_test-run">Prova</a> </li></ul></li><li> <a href="#how-to-cite">Come citare</a> </li><li> <a href="#authors-and-reviewers">Autori e revisori</a> </li><li> <a href="#license">Licenza</a> </li><li> <a href="#acknowledgement">Riconoscimento</a> </li></ul><h2> <a class="anchor" id="in-a-glance" href="#in-a-glance"><i class="fa fa-link"></i></a> In uno sguardo </h2><p> Questo modulo calcola i costi della fornitura di calore negli edifici tramite tecnologie di fornitura di calore decentralizzate. Gli input per il modulo sono i costi di investimento, i costi di O&amp;M, i prezzi dell&#39;energia, il profilo di carico orario della domanda di calore, nonché il tempo di ammortamento e il tasso di interesse. Inoltre, sono richieste la composizione degli edifici e gli stati di ristrutturazione degli edifici nell&#39;area analizzata. I risultati sono i costi di fornitura di calore di varie tecnologie di fornitura di calore decentralizzata per gli edifici definiti e le ipotesi di prezzo, nonché i costi di fornitura decentralizzata di riferimento per l&#39;area analizzata e le relative emissioni di CO2. </p><p> <a href="#table-of-contents"><strong><code>To Top</code></strong></a> </p> <h2> <a class="anchor" id="introduction" href="#introduction"><i class="fa fa-link"></i></a> introduzione </h2><p> Questo modulo di calcolo calcola il costo livellato del calore (LCOH) per varie tecnologie di fornitura di calore decentralizzate per fornire calore a un edificio definito dall&#39;utente. </p><img alt="" src="https://raw.githubusercontent.com/HotMaps/hotmaps_wiki/master/Images/cm_decentral_heating/new/1.png"/><p> Le tecnologie sono ad esempio le seguenti: </p><ul><li> Pompa di calore aria-aria, aria-acqua e salamoia-acqua, </li><li> Stufa elettrica, </li><li> Caldaia a bio-olio, </li><li> Caldaia a olio, </li><li> Biomasse automatiche e manuali, </li><li> Stufa a legna, </li><li> Gas naturale, </li><li> Solare termico </li></ul><p><ins> <code><strong><a href="#table-of-contents">To Top</a></strong></code> </ins> </p><h2> <a class="anchor" id="inputs-and-outputs-" href="#inputs-and-outputs-"><i class="fa fa-link"></i></a> Ingressi e uscite: </h2><h3> <a class="anchor" id="inputs" href="#inputs"><i class="fa fa-link"></i></a> Ingressi </h3><h4> <a class="anchor" id="default-input-data" href="#default-input-data"><i class="fa fa-link"></i></a> Dati di input predefiniti </h4><p> Parte dei dati di input proviene dai set di dati predefiniti di Hotmaps. Questi includono: </p><ul><li> NUTS Raster </li><li> Patrimonio edilizio (Livello NUTS0): domanda di energia utile per classe di edificio e periodo di costruzione </li><li> Profili di carico (livello NUTS2) </li><li> Dati finanziari - Sistemi di riscaldamento (livello NUTS0) </li><li> Dati finanziari - Prezzi dei vettori energetici (livello NUTS0) </li></ul><p> <a href="#table-of-contents"><strong><code>To Top</code></strong></a> </p> <h4> <a class="anchor" id="manual-input-data" href="#manual-input-data"><i class="fa fa-link"></i></a> Dati di input manuale </h4><p> I seguenti dati di input devono essere inseriti manualmente dopo la selezione di un&#39;area: </p><ul><li> Età dell&#39;edificio, </li><li> Tasso d&#39;interesse, </li><li> Superficie lorda, </li><li> Tipo di edificio, </li><li> Anno (per dati finanziari), </li><li> Risparmio nel riscaldamento degli ambienti, </li><li> Fattori di emissione per vettore energetico. </li></ul><p> <strong>NOTA IMPORTANTE</strong> In Hotmaps le emissioni di biomassa non sono assunte a zero perché Hotmaps è uno strumento finalizzato a supportare la transizione energetica in corso. Considerare la biomassa come una fonte di energia a emissioni zero implica che la combustione della biomassa riduce le emissioni totali attuali rispetto alla combustione di combustibili fossili, il che non è vero. Lo stoccaggio dell&#39;anidride carbonica negli alberi avviene nell&#39;arco di decenni, mentre il suo rilascio nell&#39;atmosfera avviene tutto in una volta. Pertanto, si può concludere che il bilancio netto del carbonio della biomassa è zero nel lungo periodo, ma non nel breve periodo e le misure di mitigazione del cambiamento climatico devono portare risultati nella riduzione delle emissioni di carbonio nel breve periodo. </p><p><ins> <code><strong><a href="#table-of-contents">To Top</a></strong></code> </ins> </p><h3> <a class="anchor" id="outputs-" href="#outputs-"><i class="fa fa-link"></i></a> Uscite: </h3><h4> <a class="anchor" id="indicators-" href="#indicators-"><i class="fa fa-link"></i></a> Indicatori: </h4><p> Gli indicatori sono composti da un riepilogo dei parametri di input e dei valori calcolati: </p><ul><li> Codice NUTS corrispondente: codice NUTS3 della regione selezionata (quando si utilizza il livello di ettaro o il livello LAU), </li><li> risparmio inserito nel riscaldamento degli ambienti, </li><li> entrata superficie lorda, </li><li> anno inserito, </li><li> tasso di interesse inserito, </li><li> entrato in età edilizia, </li><li> tipo di edificio inserito, </li><li> fabbisogno energetico annuo utile dell&#39;edificio, </li><li> Carico termico, </li><li> Settore, </li><li> Tipo di edificio utilizzato per ogni dato finanziario. </li></ul><p><ins> <code><strong><a href="#table-of-contents">To Top</a></strong></code> </ins> </p><h4> <a class="anchor" id="graphs-" Domanda finale di energia </li><li> Costi totali </li><li> Fattore di rendita </li><li> Carico di calore </li></ul><p><ins> <code><strong><a href="#table-of-contents">To Top</a></strong></code> </ins> </p><h2> <a class="anchor" id="methodology-" href="#methodology-"><i class="fa fa-link"></i></a> Metodologia: </h2><p> In base alla regione selezionata vengono determinate le regioni NUTS0 e NUTS2 corrispondenti. </p><p> Quindi, per il tipo di edificio selezionato, l&#39;età dell&#39;edificio e la regione NUTS0 viene selezionata la domanda di energia utile specifica e la domanda di calore Sulla base del fabbisogno termico annuo dell&#39;edificio e dei risparmi immessi nel riscaldamento degli ambienti, il carico termico viene calcolato utilizzando i profili di carico termico. </p><img energetici. </p><img alt="" src="https://raw.githubusercontent.com/HotMaps/hotmaps_wiki/master/Images/cm_decentral_heating/new/7.png"/><p> Ora abbiamo tutti i dati necessari e possiamo calcolare il
