Το πρόβλημα δείχνει το σφάλμα και ο προγραμματιστής πρέπει να το διορθώσει και να επαναφέρει το έργο του στο Git έτσι ώστε το σενάριο να μπορεί να προσπαθήσει να το ενσωματώσει ξανά το επόμενο βράδυ. </p><p> Ο πηγαίος κώδικας του σεναρίου ενσωμάτωσης είναι διαθέσιμος σε αυτόν τον σύνδεσμο: <a href="https://github.com/HotMaps/CI_DatasetIntegration">Ενσωμάτωση δεδομένων</a> </p><p><ins> <code><strong><a href="#table-of-contents">To Top</a></strong></code> </ins> </p><h2> <a class="anchor" id="add-your-datasets-on-gitlab" href="#add-your-datasets-on-gitlab"><i class="fa fa-link"></i></a> Προσθέστε τα σύνολα δεδομένων σας στο GitLab </h2><p> Για να προσθέσετε σύνολα δεδομένων στη λίστα δημόσιων συνόλων δεδομένων, πρέπει να προωθηθούν σε ένα νέο αποθετήριο Git στο GitLab. Εδώ είναι ο Οργανισμός GitLab όπου θα πρέπει να πιέζονται τα σύνολα δεδομένων: <a href="https://gitlab.com/hotmaps">Δεδομένα στο GitLab</a> . </p><p> Μια φορά την ημέρα οι αποθήκες ελέγχονται για νέες δεσμεύσεις και ενσωματώνονται αν ναι. Η διαδικασία ενσωμάτωσης ελέγχει αν τα δεδομένα συμμορφώνονται με τις προδιαγραφές ή όχι. </p><p> Εδώ είναι η προδιαγραφή: <a href="uploads/Hotmaps_Data-upload-on-Gitlab_2017-12-04_V4.pdf">Hotmaps_Data-upload-on-Gitlab_2017-12-04_V4.pdf</a> </p><p><ins> <code><strong><a href="#table-of-contents">To Top</a></strong></code> </ins> </p><h2> <a class="anchor" id="list-of-main-repositories" href="#list-of-main-repositories"><i class="fa fa-link"></i></a> Κατάλογος των κύριων αποθετηρίων </h2><p> Ο κώδικας της εφαρμογής βρίσκεται στο GitHub κάτω από την <a href="https://github.com/HotMaps">οργάνωση Hotmaps</a> . Αυτή η οργάνωση διαθέτει αρκετά αποθετήρια </p><ul><li> <a href="https://github.com/HotMaps/Hotmaps-toolbox-service">Το Hotmaps-toolbox-client</a> περιέχει το μέτωπο της εφαρμογής μας. Πρόκειται για ένα γωνιακό έργο (JavaScript) </li><li> <a href="https://github.com/HotMaps/Hotmaps-toolbox-service">Η υπηρεσία Hotmaps-toolbox-service</a> περιέχει το API για την εφαρμογή μας. Βασίζεται στο Flask (Python) </li><li> <a href="https://github.com/HotMaps/hotmaps_wiki">Το Hotmaps-wiki</a> είναι το Wiki που διαβάζετε αυτήν τη στιγμή </li><li> <a href="https://github.com/HotMaps/base_calculation_module">βασική μονάδα υπολογισμού</a> είναι το βασικό πρότυπο που μπορείτε να χρησιμοποιήσετε για να δημιουργήσετε τις δικές σας μονάδες υπολογισμού για τα Hotmaps </li><li> μια λίστα υπομονάδων υπολογισμού </li></ul><p><ins> <code><strong><a href="#table-of-contents">To Top</a></strong></code> </ins> </p><h2> <a class="anchor" id="how-to-contribute-code" href="#how-to-contribute-code"><i class="fa fa-link"></i></a> Πώς να συμβάλλετε στον κώδικα </h2><p> Αν θέλετε να προσθέσετε κάποιο κωδικό στα Hotmaps, έχετε δύο δυνατότητες: εάν θέλετε να ενημερώσετε απευθείας τη διεπαφή ή το backend, πρέπει να τροποποιήσετε τον αποθετήριο πελάτη ή υπηρεσία της εργαλειοθήκης. Αν θέλετε να προσθέσετε τη δική σας υπομονάδα υπολογισμού, μπορείτε να δημιουργήσετε το δικό σας αποθετήριο ακολουθώντας <a href="https://github.com/HotMaps/base_calculation_module">το readme του repository base_calculation_module</a> </p><p> Εάν θέλετε να εκτελέσετε κάποια εργασία στο αποθετήριο Git, παρακαλούμε μην εργάζεστε άμεσα με τον κύριο κατάλογο. Δημιουργήστε ένα νέο υποκατάστημα από τον κλάδο ανάπτυξης, κάντε την εργασία σας σε αυτόν τον κλάδο και όταν δοκιμάζεται η λειτουργία σας, μπορείτε να συγχωνεύσετε την εργασία σας με το αναπτυσσόμενο υποκατάστημα ως εμφάνιση στο παρακάτω γράφημα. </p><p><img alt="git_workflow" src="en-images/git_workflow.png"/></p><p> Για να προωθήσετε κάτι σε κάποιο χώρο αποθήκευσης Hotmaps πρέπει να είστε μέλος της ομάδας Hotmaps, αν δεν είστε ακόμα σε θέση να εκτελέσετε μια πιρούνι του εργαλείου μας για να αναπτύξετε το δικό σας εργαλείο. </p><p> Μπορείτε να βρείτε περισσότερες πληροφορίες σχετικά με τον τρόπο εργασίας σε αυτά τα έγγραφα: </p><ul><li> <a href="uploads/Hotmaps_python_best_practices_tutorial_2017-08-07_v01.pdf">Hotmaps_python_best_practices_tutorial_2017-08-07_v01.pdf</a> </li><li> <a href="uploads/Hotmaps_Testing_in_python_tutorial_pytest_2017-08-07_v01.pdf">Hotmaps_Testing_in_python_tutorial_pytest_2017-08-07_v01.pdf</a> </li><li> <a href="uploads/GitFlow_Guidelines_CREM_2017-02-01.pdf">GitFlow_Guidelines_CREM_2017-02-01.pdf</a> </li></ul><p><ins> <code><strong><a href="#table-of-contents">To Top</a></strong></code> </ins> </p><h2> <a class="anchor" id="description-of-it-infrastructure" href="#description-of-it-infrastructure"><i class="fa fa-link"></i></a> Περιγραφή της υποδομής πληροφορικής </h2><p><img alt="ReverseProxy_architecture_latest" src="en-images/ReverseProxy_architecture_latest.png"/></p><p> Όλες οι υπηρεσίες και τα εξαρτήματα χρησιμοποιούνται μέσω του δικού τους δοχείου Docker. Όλα αυτά τα δοχεία ορίζονται σε ένα αρχείο αρχείου docker-compose. Η παραπάνω εικόνα αντιπροσωπεύει την αρχιτεκτονική πληροφορικής των Hotmaps. </p><p> Ορισμένες οργανώσεις-εταίροι περιορίζουν την επικοινωνία μόνο στη θύρα 80. Για να αποφευχθούν τα προβλήματα που προκαλούνται από αυτόν τον περιορισμό, δημιουργήθηκε μια αντίστροφη εξουσιοδότηση. Αυτός ο αντιστρεπτός διακομιστής μεσολάβησης προσφέρει ένα μόνο σημείο εισόδου και στη συνέχεια διανέμει το αίτημα που αποστέλλει ο πελάτης στην οικεία υπηρεσία. Το αντίστροφο πληρεξούσιο αποτελείται από τρία στοιχεία: </p><ol><li> Αντίστροφη διακομιστής μεσολάβησης: χρησιμεύει ως μοναδικό σημείο εισόδου και διανέμει αιτήματα στις σωστές υπηρεσίες. </li><li> Proxy-gen: είναι μια υπηρεσία που χαρτογραφεί αυτόματα όλες τις υπηρεσίες στον αντίστροφο διακομιστή μεσολάβησης. Επομένως, δεν είναι απαραίτητο να προσθέσετε με μη αυτόματο τρόπο μια νέα υπηρεσία στη διαμόρφωση διακομιστή μεσολάβησης </li><li> επιτρέπει-κρυπτογράφηση: είναι μια υπηρεσία που επιτρέπει τη χρήση του πρωτοκόλλου SSL. Είναι απαραίτητο για να ενεργοποιήσετε το πρωτόκολλο https. Τα πιστοποιητικά SSL υπογράφονται από μια διεύθυνση ηλεκτρονικού ταχυδρομείου που έχει διαμορφωθεί σε αυτήν την υπηρεσία. </li></ol><p> Υπάρχουν τρία δίκτυα: </p><ul><li> Το hotmaps_nginx επιτρέπει στο αντίστροφο proxy να επικοινωνεί με το api, το frontend και το geoserver. Επιτρέπει κυρίως τη διανομή ενός αιτήματος στη σωστή υπηρεσία μεταξύ των τριών. </li><li> Το hotmaps_backend επιτρέπει την επικοινωνία μεταξύ όλων των στοιχείων του backend: api, frontend, geoserver και της βάσης δεδομένων PostgreSQL. </li><li> Το hotmaps_cm-net επιτρέπει την επικοινωνία μεταξύ κάθε υπομονάδας υπολογισμού και του api. </li></ul><p> Κάθε υπομονάδα υπολογισμού έχει το δικό της δοχείο Docker. </p><h3> <a class="anchor" id="run-with-docker" href="#run-with-docker"><i class="fa fa-link"></i></a> Εκτέλεση με το Docker </h3><p> Hotmaps χρησιμοποιεί <a href="https://www.docker.com/">Docker</a> λογισμικού και <a href="https://docs.docker.com/compose/">Docker-Σύνθεση</a> εργαλείο για τη διαχείριση των εμπορευματοκιβωτίων. Ένα αρχείο docker-compose.yml περιέχει ολόκληρη τη διαμόρφωση της αρχιτεκτονικής του Docker (config σε δοχεία, δίκτυα, συνδέσμους, ...). Αυτό επιτρέπει την εκτέλεση των εμπορευματοκιβωτίων με μία απλή εντολή: </p><pre> <code class="language-shell">docker-compose up</code> </pre><p> <em>Υπάρχουν περισσότερα σχετικά με τη σύνθεση του docker στο webiste του Docker: <a href="https://docs.docker.com/compose/reference/">Σύνταξη αναφοράς γραμμής εντολών</a> και <a href="https://docs.docker.com/compose/compose-file/">Σύνταξη αναφοράς αρχείου</a> .</em> </p><p> Υπάρχει μόνο ένα κιβώτιο που εκτελείται ξεχωριστά από άλλα: είναι η βάση δεδομένων, επειδή πρέπει να παραμείνει συνεχώς επάνω. Γι &#39;αυτό δεν είναι στο αρχείο διαμόρφωσης docker-compose. </p><h3> <a class="anchor" id="server-infrastructure" href="#server-infrastructure"><i class="fa fa-link"></i></a> Υποδομή διακομιστή </h3><h4> <a class="anchor" id="infrastructure" href="#infrastructure"><i class="fa fa-link"></i></a> Υποδομή </h4><p> Προς το παρόν, ο διακομιστής φιλοξενείται στο HES-SO στην Ελβετία. Υπάρχουν 2 διαθέσιμα μηχανήματα: ένα για ανάπτυξη (ανάπτυξη και δοκιμή) και ένα για παραγωγή (η πραγματική εργαλειοθήκη, διαθέσιμη στη διεύθυνση <a href="https://www.hotmaps.eu">www.hotmaps.eu</a> ). </p><p> Και τα δύο μηχανήματα έχουν την ίδια προδιαγραφή: </p><ul><li> CPU: Intel Xeon E5-2680 v4 (8) @ 2,4 GHz) </li><li> RAM: 16GB </li><li> Σκληρός δίσκος: 500GB </li><li> OS: Ubutnu 16.04 LTS </li></ul><h4> <a class="anchor" id="performance" href="#performance"><i class="fa fa-link"></i></a> Εκτέλεση </h4><p> Εκτελούμε συχνά δοκιμές απόδοσης στον εξυπηρετητή ανάπτυξης, προκειμένου να εγγυηθούμε έναν ορισμένο αριθμό ταυτόχρονων χρηστών. </p><p> Για παράδειγμα, παρακάτω είναι τα αποτελέσματα της πρώτης έκδοσης beta έναντι των μελλοντικών δοκιμών απελευθέρωσης.
